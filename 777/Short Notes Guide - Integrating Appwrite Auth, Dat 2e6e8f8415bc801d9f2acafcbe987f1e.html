1. Setting up Appwrite Project

Set up your Appwrite backend before coding.

1.1. Creating a New Appwrite Project

  1. Install Appwrite: Install Appwrite (see https://appwrite.io/docs/installation).
  1. Access Console: Open your Appwrite console (e.g., http://localhost).
  1. Create Project: Create a project (e.g., "MovieListingApp"). Note its Project ID.

1.2. Configuring Platforms

Inform Appwrite about connecting platforms.

  1. Add Platform: Go to "Overview" -> "Add Platform".
  1. Select "Flutter App" or "React Native App": Choose "Flutter App" or a generic "Client" for React Native.
  1. Enter Application ID: Use your React Native app's package name (e.g., com.yourcompany.movielisting), found in app.json or package.json.

1.3. Creating Authentication

Appwrite handles user authentication.

  1. Users: Find registered users under "Auth" -> "Users" in the console.
  1. Settings: Configure authentication options (e.g., email verification, password policies, OAuth) in "Auth" -> "Settings". Email/password is the current focus.

1.4. Creating Database and Collections

Appwrite Database stores user watchlists.

  1. Create Database: Go to "Database", "Create Database" (e.g., "MovieData"). Note Database ID.
  1. Create Collection (Watchlist):
    • In the database, "Create Collection", name it "Watchlist". Note Collection ID.
    • Permissions: Crucial.
      • Read/Write Access: Grant "Document Permissions" to "users".
      • Important: For "Watchlist" collection, set "Permissions" to allow "create", "read", "update", "delete" for users. Document-level permissions in code will restrict users to their own items.
  1. Add Attributes to Watchlist Collection:
    • In "Watchlist" collection, go to "Attributes".
    • Add:
      • userId: String, true (Links to user).
      • itemId: String, true (Unique ID: movie, TV show, season, episode).
      • itemTitle: String, true (Title: movie, TV show, season, episode).
      • itemPosterPath: String, false (Poster/thumbnail URL).
      • itemReleaseDate: String, false (Release/air date).
      • mediaType: String, true (e.g., 'movie', 'tv', 'season', 'episode').
      • parentItemId: String, false (Parent ID for season/episode).
      • seasonNumber: Integer, false (Season number).
      • episodeNumber: Integer, false (Episode number).

2.2. Appwrite Client Initialization

Create an Appwrite client configuration file.

  1. Create src/api/appwrite.js:
    // src/api/appwrite.js
import { Client, Account, Databases } from 'appwrite';

export const appwriteConfig = {
  endpoint: process.env.APPWRITE_ENDPOINT || 'YOUR_APPWRITE_ENDPOINT', // e.g., '<http://localhost/v1>'
  projectId: process.env.APPWRITE_PROJECT_ID || 'YOUR_APPWRITE_PROJECT_ID',
  databaseId: process.env.APPWRITE_DATABASE_ID || 'YOUR_APPWRITE_DATABASE_ID',
  watchlistCollectionId: process.env.APPWRITE_WATCHLIST_COLLECTION_ID || 'YOUR_APPWRITE_WATCHLIST_COLLECTION_ID',
};

const client = new Client()
  .setEndpoint(appwriteConfig.endpoint)
  .setProject(appwriteConfig.projectId);

const account = new Account(client);
const databases = new Databases(client);

export { client, account, databases };

    Note: Replace placeholders with your Appwrite console values. Use environment variables (process.env.APPWRITE_...) for security.

// screens/SignUpScreen.js (example)
import React, { useState } from 'react';
import { View, Text, TextInput, Button, Alert } from 'react-native';
import AuthService from '../src/auth/authService';

const SignUpScreen = ({ navigation }) => {

  const [password, setPassword] = useState('');
  const [name, setName] = useState('');

      Alert.alert('Error', error.message || 'Something went wrong during registration.');

    <View style={{ flex: 1, justifyContent: 'center', padding: 20 }}>
      <Text style={{ fontSize: 24, marginBottom: 20 }}>Sign Up</Text>
      <TextInput
        placeholder="Name"
        value={name}
        onChangeText={setName}
        style={{ borderWidth: 1, borderColor: 'gray', padding: 10, marginBottom: 10 }}
      />
      <TextInput
        placeholder="Email"
        value={email}
        onChangeText={setEmail}
        keyboardType="email-address"
        autoCapitalize="none"
        style={{ borderWidth: 1, borderColor: 'gray', padding: 10, marginBottom: 10 }}
      />
      <TextInput
        placeholder="Password"
        value={password}
        onChangeText={setPassword}
        secureTextEntry
        style={{ borderWidth: 1, borderColor: 'gray', padding: 10, marginBottom: 20 }}
      />
      <Button title={loading ? 'Signing Up...' : 'Sign Up'} onPress={handleSignUp} disabled={loading} />

4. Implementing Appwrite Database for Watchlist

Handle logic for "Watchlist" collection interaction.

4.1. Creating Watchlist Collection (Done in Appwrite Console)

(As in 1.4) Ensure "Watchlist" collection has userId, itemId, itemTitle, itemPosterPath, itemReleaseDate, mediaType, parentItemId, seasonNumber, and episodeNumber.

4.2. Adding an Item to Watchlist

Create src/database/watchlistService.js:

// src/database/watchlistService.js
import { ID, Permission, Role } from 'appwrite';
import { databases, appwriteConfig } from '../api/appwrite';

const WatchlistService = {
  async addItemToWatchlist(userId, itemData) {
    try {
      const { itemId, itemTitle, itemPosterPath, itemReleaseDate, mediaType, parentItemId, seasonNumber, episodeNumber } = itemData;
      const document = await databases.createDocument(
        appwriteConfig.databaseId,
        appwriteConfig.watchlistCollectionId,
        ID.unique(),
        {
          userId,
          itemId,
          itemTitle,
          itemPosterPath,
          itemReleaseDate,
          mediaType,
          parentItemId,
          seasonNumber,
          episodeNumber,
        },
        [
          Permission.read(Role.user(userId)),
          Permission.write(Role.user(userId)),
          Permission.update(Role.user(userId)),
          Permission.delete(Role.user(userId)),
        ]
      );
      console.log('Item added to watchlist:', document);
      return document;
    } catch (error) {
      console.error('Error adding item to watchlist:', error);
      throw error;
    }
  },

  // ... other watchlist methods
};

export default WatchlistService;

Permissions Explanation:
Document-level permissions (read, write, delete for Role.user(userId)) ensure users manage only their watchlist items, overriding broader collection permissions.

4.3. Removing an Item from Watchlist

Add to src/database/watchlistService.js:

// src/database/watchlistService.js (continued)
import { Query } from 'appwrite';
// ...
const WatchlistService = {
  // ... addItemToWatchlist method

  async removeItemFromWatchlist(userId, itemId, mediaType) {
    try {
      // First, find the document ID for the item in the user's watchlist
      const response = await databases.listDocuments(
        appwriteConfig.databaseId,
        appwriteConfig.watchlistCollectionId,
        [
          Query.equal('userId', userId),
          Query.equal('itemId', itemId),
          Query.equal('mediaType', mediaType), // Important for uniqueness
        ]
      );

      if (response.documents.length > 0) {
        const documentId = response.documents[0].$id;
        await databases.deleteDocument(
          appwriteConfig.databaseId,
          appwriteConfig.watchlistCollectionId,
          documentId
        );
        console.log('Item removed from watchlist:', documentId);
        return true;
      } else {
        console.warn('Item not found in watchlist for this user.');
        return false;
      }
    } catch (error) {
      console.error('Error removing item from watchlist:', error);
      throw error;
    }
  },

  // ... other watchlist methods
};

export default WatchlistService;

4.5. Checking if an Item is in Watchlist

Add to src/database/watchlistService.js:

// src/database/watchlistService.js (continued)
import { Query } from 'appwrite';
// ...
const WatchlistService = {
  // ... addItemToWatchlist, removeItemFromWatchlist, getWatchlist methods

  async isItemInWatchlist(userId, itemId, mediaType) {
    try {
      const response = await databases.listDocuments(
        appwriteConfig.databaseId,
        appwriteConfig.watchlistCollectionId,
        [
          Query.equal('userId', userId),
          Query.equal('itemId', itemId),
          Query.equal('mediaType', mediaType), // Important for uniqueness
          Query.limit(1), // We only need to know if at least one exists
        ]
      );
      return response.documents.length > 0;
    } catch (error) {
      console.error('Error checking item in watchlist:', error);
      throw error;
    }
  },
};

export default WatchlistService;

5. React Context API for Global State Management

Context API manages global states (auth, watchlist) without prop-drilling.

5.1. Understanding Context API

  • createContext: Creates a Context object.
  • Provider: Component supplying context value to descendants.
  • useContext: Hook to read current context value.

5.2. Creating an Auth Context

This context manages the current user object and authentication state (login, logout).

Create src/context/AuthContext.js:

// src/context/AuthContext.js
import React, { createContext, useContext, useState, useEffect } from 'react';
import AuthService from '../auth/authService';

const AuthContext = createContext(null);

export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const checkUser = async () => {
      try {
        const currentUser = await AuthService.getCurrentUser();
        setUser(currentUser);
      } catch (error) {
        console.error('Failed to fetch current user on mount:', error);
        setUser(null);
      } finally {
        setLoading(false);
      }
    };
    checkUser();
  }, []);

  const login = async (session) => {
    // After successful login from AuthService, fetch user details
    const currentUser = await AuthService.getCurrentUser();
    setUser(currentUser);
  };

  const logout = async () => {
    try {
      await AuthService.logout();
      setUser(null);
    } catch (error) {
      console.error('Logout failed:', error);
      throw error;
    }
  };

  return (
    <AuthContext.Provider value={{ user, loading, login, logout, isAuthenticated: !!user }}>
      {children}
    </AuthContext.Provider>
  );
};

export const useAuth = () => {
  const context = useContext(AuthContext);
  if (!context) {
    throw new Error('useAuth must be used within an AuthProvider');
  }
  return context;
};

5.3. Creating a Watchlist Context

This context manages the user's watchlist (add, remove, check items).

Create src/context/WatchlistContext.js:

// src/context/WatchlistContext.js
import React, { createContext, useContext, useState, useEffect, useCallback } from 'react';
import WatchlistService from '../database/watchlistService';
import { useAuth } from './AuthContext'; // To get the current user ID

const WatchlistContext = createContext(null);

export const WatchlistProvider = ({ children }) => {
  const { user, isAuthenticated, loading: authLoading } = useAuth();
  const [watchlist, setWatchlist] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  const fetchWatchlist = useCallback(async () => {
    if (!isAuthenticated || !user?.$id) {
      setWatchlist([]);
      setLoading(false);
      return;
    }

    setLoading(true);
    setError(null);
    try {
      const items = await WatchlistService.getWatchlist(user.$id);
      setWatchlist(items);
    } catch (err) {
      console.error('Failed to fetch watchlist:', err);
      setError(err);
    } finally {
      setLoading(false);
    }
  }, [isAuthenticated, user?.$id]);

  useEffect(() => {
    if (!authLoading) { // Only fetch watchlist once auth state is resolved
      fetchWatchlist();
    }
  }, [authLoading, fetchWatchlist]);

  const addItemToWatchlist = async (itemData) => {
    if (!isAuthenticated || !user?.$id) {
      throw new Error('User not authenticated.');
    }
    setLoading(true); // Indicate action is in progress
    try {
      const newItem = await WatchlistService.addItemToWatchlist(user.$id, itemData);
      setWatchlist((prev) => [newItem, ...prev]); // Add new item to the top
      return newItem;
    } catch (err) {
      setError(err);
      throw err;
    } finally {
      setLoading(false);
    }
  };

  const removeItemFromWatchlist = async (itemId, mediaType) => {
    if (!isAuthenticated || !user?.$id) {
      throw new Error('User not authenticated.');
    }
    setLoading(true); // Indicate action is in progress
    try {
      const success = await WatchlistService.removeItemFromWatchlist(user.$id, itemId, mediaType);
      if (success) {
        setWatchlist((prev) => prev.filter((item) => !(item.itemId === itemId && item.mediaType === mediaType)));
      }
      return success;
    } catch (err) {
      setError(err);
      throw err;
    } finally {
      setLoading(false);
    }
  };

  const isItemInWatchlist = useCallback((itemId, mediaType) => {
    return watchlist.some((item) => item.itemId === itemId && item.mediaType === mediaType);
  }, [watchlist]);

  return (
    <WatchlistContext.Provider
      value={{
        watchlist,
        loading,
        error,
        addItemToWatchlist,
        removeItemFromWatchlist,
        isItemInWatchlist,
        refreshWatchlist: fetchWatchlist, // Allow components to manually refresh
      }}
    >
      {children}
    </WatchlistContext.Provider>
  );
};

export const useWatchlist = () => {
  const context = useContext(WatchlistContext);
  if (!context) {
    throw new Error('useWatchlist must be used within a WatchlistProvider');
  }
  return context;
};

5.4. Combining Context Providers

Wrap your app with AuthProvider and WatchlistProvider. WatchlistProvider nests inside AuthProvider due to its dependency.

Modify your App.js:

// App.js
import React from 'react';
import AppNavigator from './src/navigation/AppNavigator'; // Your main navigator
import { AuthProvider } from './src/context/AuthContext';
import { WatchlistProvider } from './src/context/WatchlistContext';
import { StatusBar } from 'react-native';

export default function App() {
  return (
    <AuthProvider>
      <WatchlistProvider>
        <StatusBar barStyle="light-content" />
        <AppNavigator />
      </WatchlistProvider>
    </AuthProvider>
  );
}

  const [loading, setLoading] = useState(true);

      setLoading(true);

        let details;

          details = await fetchMovieDetails(itemId);

          details = await fetchTvShowDetails(itemId);

        setItemDetails(details);
      } catch (error) {
        console.error('Error fetching item details:', error);
        Alert.alert('Error', 'Failed to fetch item details.');
      } finally {
        setLoading(false);
      }

    getDetails();

    if (!isAuthenticated) {
      Alert.alert('Login Required', 'Please log in to manage your watchlist.');
      return;
    }

    setIsAddingRemoving(true);
    try {
      if (isInWatchlist) {
        await removeItemFromWatchlist(String(itemId), mediaType);
        Alert.alert('Removed', `${itemDetails?.title || itemDetails?.name} removed from watchlist.`);
      } else {
        const itemData = {
          itemId: String(itemId),
          itemTitle: itemDetails?.title || itemDetails?.name || `Season ${seasonNumber} Episode ${episodeNumber}`, // Adjust based on mediaType
          itemPosterPath: itemDetails?.poster_path || itemDetails?.still_path,

          episodeNumber: episodeNumber,
        };
        await addItemToWatchlist(itemData);
        Alert.alert('Added', `${itemDetails?.title || itemDetails?.name || `Season ${seasonNumber} Episode ${episodeNumber}`} added to watchlist!`);
      }
    } catch (error) {
      Alert.alert('Error', error.message || 'Could not update watchlist.');
      console.error('Watchlist update error:', error);
    } finally {
      setIsAddingRemoving(false);
    }
  };

  if (loading || (!itemDetails && !error)) {
    return (
      <View style={styles.loadingContainer}>
        <ActivityIndicator size="large" color="#0000ff" />
      </View>
    );

  return (

          style={styles.poster}
        />

        <Text style={styles.title}>{itemDetails.title || itemDetails.name}</Text>
        <Text style={styles.overview}>{itemDetails.overview || itemDetails.description}</Text>
        <Text style={styles.info}>
          Release Date: {itemDetails.release_date || itemDetails.first_air_date || itemDetails.air_date}
        </Text>
        {itemDetails.vote_average && <Text style={styles.info}>Rating: {itemDetails.vote_average?.toFixed(1)} / 10</Text>}
        {mediaType === 'season' && seasonNumber && <Text style={styles.info}>Season: {seasonNumber}</Text>}
        {mediaType === 'episode' && episodeNumber && <Text style={styles.info}>Episode: {episodeNumber}</Text>}

        },

7. Final Thoughts and Best Practices

  • Error Handling: Implement robust error handling with user-friendly messages.
  • Loading States: Use loading indicators for better UX.
  • Security:
    • Environment Variables: Use environment variables for sensitive data (e.g., Appwrite Project ID, Endpoint).
    • Appwrite Permissions: Verify Appwrite collection and document permissions; document-level permissions in watchlistService.js are crucial.
  • Offline Support: Consider local caching (e.g., AsyncStorage) for offline watchlist access.
  • Real-time Updates: Explore Appwrite's real-time capabilities for watchlist updates.
  • Scalability: Use Appwrite functions/webhooks for server-side logic as the app grows.
  • UI/UX: Focus on smooth UX and immediate feedback.

This guide provides a foundation for Appwrite auth, database (watchlist), and React Context API in your movie listing app. Adapt examples to your UI/structure.
